算法图解

1.1 算法无处不在

当我们听到“算法”这个词时，很容易想到复杂的数学公式、计算机代码、人工智能模型或者竞赛题。可是，如果把算法理解得更朴素一点，它其实并不神秘。算法本质上就是解决问题的一组明确步骤。

只要一个问题可以被拆成“先做什么、再做什么、最后得到什么结果”，它就可以被看作一个算法过程。比如查字典、整理扑克牌、排队买饭、规划路线、安排复习计划，这些看似普通的日常行为背后，都有算法思想。

例子：查字典

假设我们想在一本按拼音排序的字典里查找“明”字。最笨的方法是从第一页开始一页一页翻，直到找到为止。

更聪明的方法是先翻到中间，看当前位置的拼音是在目标之前还是之后。如果目标在前面，就继续查前半部分；如果目标在后面，就继续查后半部分。

这样每次都能排除一半的查找范围。这其实就是一种“二分查找”的思想：每次都把问题规模缩小一半。

查字典过程可以整理成下面几步：

  1. 先确定目标字的拼音。
  2. 打开字典中间位置。
  3. 比较当前位置和目标位置。
  4. 如果目标在前半部分，就继续查前半部分。
  5. 如果目标在后半部分，就继续查后半部分。
  6. 重复这个过程，直到找到目标。

生活中还有很多类似的算法思想。例如，整理扑克牌时会用到排序思想；规划最短路线时会用到图算法；安排学习任务时会用到优先级策略；做选择题排除错误选项时，也是在不断缩小搜索范围。

算法不是凭空出现的，它来自对现实问题的抽象。一个会使用算法思维的人，往往不是因为记住了多少代码，而是因为他能把混乱的问题拆成清楚的步骤。

1.2 算法是什么

算法可以理解为解决问题的方法说明书。它不一定非要用代码写出来，也可以用自然语言、流程图、伪代码或者数学表达式来描述。

核心理解：
算法关注的不是“我会不会写代码”，而是“我能不能把问题拆解成清楚、稳定、可以重复执行的步骤”。

一个好的算法通常有几个特点：

  • 明确性：每一步都清楚，不含糊。
  • 有限性：算法不能无限执行，必须在某个时刻结束。
  • 输入明确：需要什么数据，要提前说清楚。
  • 输出明确：最终要得到什么结果，要能判断。
  • 可重复性：同样的输入，在同样条件下应该得到同样的输出。

比如“做一道数学题”这个说法太模糊，不是严格算法；但“先读题，列变量，建立方程，求解方程，代回检验”就更接近算法。

再比如“整理房间”听起来也很模糊，但如果写成：

  1. 先把桌面物品分类。
  2. 把书放回书架。
  3. 把垃圾放进垃圾袋。
  4. 把常用物品放到手边。
  5. 最后检查一遍是否遗漏。

这就已经具有算法的雏形。

所以，算法并不只是程序员才需要学习的内容。它是一种结构化思维方式，可以帮助我们更清楚地理解问题，也可以帮助我们更高效地解决问题。

1.3 二分查找

二分查找适用于已经排好序的数据。它每次都查看中间元素，并根据比较结果决定继续查找左半部分还是右半部分。

二分查找的优势在于：每比较一次，问题规模就缩小一半。数据越大，这种优势越明显。

例子：猜数字

假设有人心里想了一个 1 到 100 之间的数字。你可以从 1 开始一个一个猜，但最多可能猜 100 次。

如果你每次都猜中间值，比如先猜 50，对方告诉你“大了”还是“小了”，你就可以马上排除一半数字。这样最多只需要很少几次就能猜中。

二分查找的基本步骤是：

  1. 设置左边界 left 和右边界 right。
  2. 取中间位置 mid。
  3. 如果中间值等于目标值，就返回结果。
  4. 如果中间值小于目标值，就去右半部分继续查找。
  5. 如果中间值大于目标值，就去左半部分继续查找。
  6. 如果查找范围为空，说明目标不存在。

Python 示例：

def binary_search(arr, target):
    left = 0
    right = len(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1


numbers = [1, 3, 5, 7, 9, 11, 13]
print(binary_search(numbers, 9))

二分查找的关键不在于代码有多复杂，而在于它背后的思想：每一步都尽量减少无用搜索。

在普通查找中，如果列表有 100 个元素，最坏情况下可能要查 100 次；而二分查找每次砍掉一半，查找次数会大大减少。

1.4 选择排序

选择排序的思想很直观：每次从未排序部分中找到最小值，把它放到当前应该放的位置。

例子：整理扑克牌

如果你手里有一堆乱序扑克牌，可以每次从剩余牌中找出最小的一张，放到左边已经排好序的区域。不断重复后，整副牌就会变成有序状态。

选择排序的步骤可以理解为：

  1. 从全部元素中找到最小值。
  2. 把这个最小值放到第一个位置。
  3. 从剩下的元素中继续找最小值。
  4. 把它放到第二个位置。
  5. 重复这个过程，直到所有元素有序。

Python 示例：

def selection_sort(arr):
    arr = arr[:]

    for i in range(len(arr)):
        min_index = i

        for j in range(i + 1, len(arr)):
            if arr[j] < arr[min_index]:
                min_index = j

        arr[i], arr[min_index] = arr[min_index], arr[i]

    return arr


data = [5, 3, 6, 2, 10]
print(selection_sort(data))

选择排序容易理解，但效率并不高。它适合用来入门排序思想，不适合处理大规模数据。

选择排序的主要问题在于：无论数据原本是否接近有序，它都需要不断扫描剩余元素寻找最小值。因此，它的时间复杂度通常是 \(O(n^2)\)。

1.5 递归思想

递归就是函数调用自己。它适合处理可以被拆成相同小问题的问题。

递归通常包含两个部分：递归出口 和 递归关系。
没有递归出口，程序就会无限调用下去。

例如，计算阶乘：

def factorial(n):
    if n == 1:
        return 1
    return n * factorial(n - 1)


print(factorial(5))

这里的递归出口是：

if n == 1:
    return 1

递归关系是：

return n * factorial(n - 1)

递归的难点在于不要试图把每一层都在脑子里完全展开，而是要相信函数本身已经能够解决更小的问题。

递归常见于树结构、分治算法、回溯算法等内容中。比如遍历文件夹、遍历树、求解迷宫路径，都可以使用递归思想。

1.6 栈与队列

栈和队列是两种常见的数据结构。

  • 栈：后进先出。
  • 队列：先进先出。

栈类似一摞盘子，最后放上去的盘子最先被拿走。队列类似排队买票，先来的人先被处理。

栈常见应用包括：

  • 函数调用。
  • 撤销操作。
  • 浏览器返回。
  • 括号匹配。
  • 深度优先搜索。

队列常见应用包括：

  • 任务排队。
  • 消息处理。
  • 打印队列。
  • 广度优先搜索。
  • 操作系统进程调度。

栈和队列本身并不复杂，但它们是理解很多算法的基础。

1.7 哈希表

哈希表通过键值映射来快速查找数据。Python 里的字典就是一种典型的哈希表结构。

student = {
    "name": "sanxi",
    "major": "信息与计算科学",
    "grade": "大二"
}

print(student["major"])

哈希表的优势是查询速度快。只要知道 key，就可以很快找到对应的 value。

不过，哈希表也需要考虑冲突问题。不同的 key 可能通过哈希函数得到相同的位置，这时候就需要额外的冲突处理方法。

哈希表在实际开发中非常常见，例如：

  • 根据用户名查找用户信息。
  • 根据商品编号查找商品价格。
  • 统计词频。
  • 去重。
  • 缓存查询结果。

1.8 图与广度优先搜索

图可以用来描述对象之间的关系，比如城市之间的路线、网页之间的链接、社交网络中的好友关系等。

图由两个部分组成：

  • 顶点：表示对象。
  • 边：表示对象之间的关系。

广度优先搜索从一个起点出发，先访问离它最近的一层节点，再访问更远的节点。

它常用于：

  • 查找最短路径。
  • 判断两个节点是否连通。
  • 社交网络关系扩散。
  • 地图路径层级搜索。
  • 推荐系统中的关系扩展。

广度优先搜索通常需要队列来辅助实现。先进入队列的节点会先被处理，这正好符合“按层扩展”的搜索方式。

1.9 Dijkstra 算法

Dijkstra 算法用于求解带非负权重图中的最短路径问题。

它的基本思想是：

  1. 从起点出发。
  2. 记录起点到每个节点的当前最短距离。
  3. 每次选择当前距离最短且未处理过的节点。
  4. 用这个节点去更新它相邻节点的距离。
  5. 重复直到所有节点处理完成。

Dijkstra 算法不适合存在负权边的情况。如果图中有负权边，需要考虑 Bellman-Ford 等其他算法。

Dijkstra 算法常见于地图导航、网络路由、路径规划等场景。

1.10 动态规划

动态规划适合解决具有重叠子问题和最优子结构的问题。

它的核心是：保存已经计算过的结果，避免重复计算。

常见例子包括：

  • 斐波那契数列。
  • 背包问题。
  • 最长公共子序列。
  • 最短路径问题。
  • 编辑距离问题。

递归可能会重复计算大量子问题，而动态规划会把这些子问题结果保存起来，后面直接使用。

动态规划的难点通常不在代码，而在于找到合适的状态表示和状态转移关系。

1.11 贪心算法

贪心算法每一步都选择当前看起来最优的方案。它实现简单，但不一定总能得到全局最优解。

例子：零钱找零

如果币值系统合理，每次选择不超过剩余金额的最大面额硬币，通常可以快速得到找零方案。

贪心算法适合那些局部最优能够推出全局最优的问题。如果局部选择会影响后续整体结构，那么贪心算法可能失败。

判断一个问题能不能使用贪心算法，关键要看局部最优选择是否能够保证整体最优。
